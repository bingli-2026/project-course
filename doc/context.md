# Project Context

## 1. 课题目标

本项目是一个为期 3 个月的 project 课程课设，目标不是单独做“视频运动放大”，而是完成一个可交付的双测点设备运行状态识别系统：

- 测点 1：三轴震动传感器
- 测点 2：全局相机

系统目标：

1. 使用三轴传感器与全局相机分别提取机器振动特征。
2. 利用双模态特征融合判断机器是否正常运行。
3. 在新增少量工况样本时，支持轻量级增量更新，而不是每次完全从头训练。

建议最终项目标题：

`基于三轴振动传感器与全局相机双测点融合的设备运行状态识别研究，并探索小样本增量更新能力`

## 2. 摄像头选型判断

当前看中的摄像头模式：

- `160x120 @ 640fps MJPEG`
- `320x240 @ 420fps MJPEG`
- `160x120 @ 420fps YUV`
- `320x240 @ 120fps YUV`

结论：

- 主实验模式优先使用 `160x120 @ 420fps YUV`
- 调试和低频实验可使用 `320x240 @ 120fps YUV`
- `MJPEG` 模式可用于探索，但不建议作为视觉测振主链路

原因：

- 本项目目标是“稳定测频”，不是“拍得更好看”
- 微小振动对压缩伪影非常敏感，`MJPEG` 会引入块效应和时域伪信号
- `YUV` 更适合后续做 ROI 时序提取、光流、亚像素位移估计和频谱分析
- 低分辨率不是致命问题，但要求目标尽量占满视野，并保证照明充足、曝光足够短、相机固定刚性足够高

工程提醒：

- `640 fps` 的理论奈奎斯特上限是 `320 Hz`
- 实际可用频段通常低于理论上限，受曝光、噪声、纹理、压缩和位移幅值影响明显
- 高频模式下必须关注快门时间与照明，否则会因为运动模糊导致频率提取失败

## 3. 技术路线定位

项目主线应当是“视觉测振 + 传感器测振 + 特征融合 + 状态识别 + 小样本增量更新”。

不建议把“运动放大”当作主路线。更合理的定位是：

- 视觉测振主链路：用于定量提取振动频率与相关特征
- 运动放大辅助链路：用于帮助肉眼确认振动区域、验证频段和展示效果

换句话说：

- `EVM / Phase-based Motion Magnification` 适合作为可视化辅助工具
- `ROI 位移时序 + FFT/STFT` 才是项目中真正应交付的测频主链路

## 4. 双测点整体方案

### 4.1 数据采集层

传感器侧：

- 三轴加速度采样：`ax, ay, az`
- 记录精确时间戳

视觉侧：

- 使用固定机位拍摄目标机器局部 ROI
- 保证 ROI 内有纹理、边缘或重复结构，便于跟踪
- 记录视频帧时间戳

关键要求：

- 两路数据必须可对齐
- 采样时必须记录统一时间基准，后续做同步与频谱对照

### 4.2 信号处理层

传感器侧：

- 去直流
- 带通滤波
- FFT / Welch PSD / STFT

视觉侧主链路：

1. 选取 ROI
2. 提取稳定跟踪点、边缘或纹理块
3. 做位移估计，得到 `dx(t)`、`dy(t)` 或单轴位移序列
4. 对位移时序做滤波与频谱分析

视觉侧候选方法按优先级排序：

1. `ROI 位移跟踪`
   - Lucas-Kanade 光流
   - 模板匹配
   - 亚像素边缘/角点跟踪
2. `亮度调制法`
   - 对固定 ROI 的灰度变化做时序分析
3. `相位法`
   - 对微小位移更敏感，但实现复杂度更高
4. `EVM / motion magnification`
   - 主要用于可视化辅助，不作为唯一测频手段

### 4.3 特征层

传感器特征建议：

- RMS
- 峰峰值
- 峭度
- 偏度
- 主频
- 前 3 个峰值频率
- 峰值幅值
- 频带能量
- 谐波比
- 频谱质心

视觉特征建议：

- 主频
- 次峰频率
- 主频幅值
- 位移 RMS
- 谐波能量比
- 频谱熵
- 频率稳定度
- 多跟踪点频谱一致性

### 4.4 融合与状态识别层

推荐使用特征级融合：

- 传感器特征向量
- 视觉特征向量
- 拼接后输入分类器

课设阶段推荐模型：

- `Random Forest`
- `XGBoost`
- `SVM`
- `MLP`

建议顺序：

1. 先做单模态基线
2. 再做双模态融合
3. 对比性能提升

优先推荐：

- 基线模型：`Random Forest` 或 `XGBoost`

原因：

- 数据量不大时更稳定
- 对特征工程友好
- 容易解释，适合课设答辩

## 5. 增量学习方案定位

3 个月课设不建议做复杂在线深度学习。增量学习建议收敛为：

`当机器新增一种工况或到来少量新样本时，模型无需从头训练，也能快速更新。`

推荐两条可行路线：

### 路线 A：经典轻量增量分类器

- `SGDClassifier`
- `Passive-Aggressive`
- 其他支持 `partial_fit` 的在线学习模型

优点：

- 实现简单
- 容易说明“增量学习”概念

缺点：

- 对小数据场景下的稳定性未必最好

### 路线 B：原型/类中心更新法

做法：

1. 固定特征提取流程
2. 为每一类状态维护一个类中心或原型向量
3. 新样本进入后更新对应类中心
4. 用最近类距离或阈值进行分类与未知状态检测

优点：

- 简单
- 可解释
- 很适合小样本新增工况
- 容易展示“未知状态”和“新工况纳入”

项目建议：

- 主分类基线：`XGBoost` 或 `Random Forest`
- 增量模块：`原型更新 / 最近类均值法`

## 6. 推荐项目范围控制

为了保证 3 个月内可交付，不建议同时追求以下目标：

- 高精度运动放大算法研发
- 复杂深度神经网络
- 在线实时部署到嵌入式硬件
- 多机器泛化
- 大规模工业异常诊断

建议收敛成果范围：

- 1 台实验设备或 1 套振动台
- 3 到 5 种状态
- 双模态采集
- 频率提取与频谱分析
- 双模态融合分类
- 小样本增量更新演示

## 7. 3 个月排期建议

### 第 1 个月

- 搭建采集系统
- 跑通相机和三轴传感器
- 完成时间同步与基础频谱分析
- 验证相机能否稳定提取主频

### 第 2 个月

- 完成视觉与传感器特征工程
- 采集正常/异常/不同转速数据
- 建立单模态与双模态基线模型
- 完成性能对比

### 第 3 个月

- 增加小样本增量更新模块
- 模拟新工况加入
- 对比更新前后识别效果
- 完成演示、图表和答辩材料

## 8. 视频技术验证策略

在设备到位前，优先进行“视频侧技术验证”，但目标要明确：

- 不是先追求最好看的运动放大效果
- 而是先验证视觉链路是否能稳定提取频率

推荐验证顺序：

1. 使用开源视频验证 ROI 时序提取与频谱分析代码
2. 使用已知频率或周期性运动视频验证主频提取是否正确
3. 再引入 motion magnification 作为辅助验证
4. 最后再迁移到真实摄像头和真实机器

## 9. 开源数据集与公开视频验证资源

以下资源更适合“视觉测振”和“视频微小运动检测”的技术验证。

### 9.1 MIT Eulerian Video Magnification 官方页面

官方页面：

- https://people.csail.mit.edu/mrub/vidmag/

适用原因：

- 是视频微小运动放大领域的经典基准资源
- 官方页面提供论文、示例和相关数据入口
- 适合先验证基础视频时序处理流程

适合优先关注的内容：

- `crane`
- `guitar`
- 其他带有微小周期性变化的视频

说明：

- 这些数据更适合做“算法链路打通”和“可视化验证”
- 与真实工业机器仍有域差异，不宜直接拿来训练最终状态分类器

### 9.2 Phase-Based Video Motion Processing 官方数据

官方页面：

- https://people.csail.mit.edu/nwadhwa/phase-video/

官方页面明确提供：

- 代码下载
- `All source and result videos`
- 工程相关示例，例如 `Engine vibrations`

适用原因：

- 相比早期线性 EVM，这套相位法对微小运动和噪声更稳
- 页面中包含更贴近机械振动的公开视频案例
- 很适合做“视觉测振可行性”验证

优先建议：

- 先下载页面中的 source videos
- 不要只看 magnified result，要优先拿原始视频跑自己的频率提取流程

### 9.3 Learning-based Video Motion Magnification / DeepMag

官方页面：

- https://cdfg.mit.edu/publications/learning-based-video-motion-magnification

该页面指向项目主页：

- http://people.csail.mit.edu/tiam/deepmag/

适用原因：

- 提供 learning-based motion magnification 的官方项目入口
- 适合后续参考 synthetic data 和学习型方法思路
- 可用于后期扩展，不建议作为第一阶段唯一依赖

说明：

- 第一阶段仍然应优先完成传统视觉测频链路
- learning-based magnification 更适合作为对比或扩展方向

## 10. 当前最建议的数据验证路线

对于本项目，视频验证建议分成两步：

### 第一阶段：链路打通

数据来源：

- MIT EVM 官方公开视频
- Phase-based 官方 source videos

验证目标：

- ROI 选取
- 位移时序提取
- FFT / Welch PSD
- 主频定位
- 多点一致性分析

输出：

- 频谱图
- 主频估计值
- 不同 ROI 的结果对比

### 第二阶段：增强与对比

数据来源：

- DeepMag 相关项目资源
- 官方 motion magnification 示例视频

验证目标：

- 对比是否使用 motion magnification 前后，频率检测更稳定还是更差
- 明确 motion magnification 在本项目中的定位是“辅助”还是“主流程一部分”

## 11. 项目中的关键风险

1. 相机高帧率模式下噪声较大，导致视觉主频提取不稳定。
2. `MJPEG` 压缩造成伪信号，影响微振检测。
3. 真实机器的纹理、光照、安装角度不利于视觉测振。
4. 双模态时间同步不准确，导致频谱对齐失败。
5. 增量学习样本过少或类别边界不清，效果不稳定。

风险应对：

- 主线优先验证 `YUV` 模式
- 先做已知频率实验，再做真实机器实验
- 视觉链路若不稳定，可降级为辅助特征与可视化模块
- 保证项目主体始终是“双测点融合状态识别”

## 12. 下一步建议

当前优先事项：

1. 先基于开源视频完成视觉测振链路验证
2. 跑通 `ROI -> 位移时序 -> 频谱 -> 主频`
3. 确认视觉测频是否稳定，再开始真实硬件实验

后续可继续补充：

- 数据下载与整理脚本
- 视频测频 Python 原型
- 与三轴传感器频谱对齐的融合脚本
